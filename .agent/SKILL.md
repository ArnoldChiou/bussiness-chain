# 專案：商業模擬遊戲 MVP (Sim Companies 風格)
# 技術棧：Vue 3 (Composition API) + Tailwind CSS + Vite (純 JavaScript)

## 🎯 開發行為與 Token 最佳化規則 (System Rules)
1. **極簡輸出**：不要解釋程式碼，不要輸出無意義的教學或問候語。直接給出可執行的程式碼。
2. **增量更新**：若進行後續修改，僅輸出有變動的程式碼區塊或函式，絕對不要每次都重印整份檔案，以減少 Token 消耗。
3. **一頁式架構 (Single-File MVP)**：為了避免上下文丟失與過度引用，MVP 階段請將所有核心 UI 與邏輯集中在 `src/App.vue` 實作。
4. **樣式規範**：嚴禁寫原生的 `<style>`，所有外觀皆必須透過 Tailwind CSS 的 class 實現。

## 🖥️ 介面設計 (UI)
使用 Tailwind CSS 建立一頁式儀表板 (Dashboard) 介面：
1. **佈局**：畫面需填滿視窗，不需滾動即可看清全局 (h-screen, grid/flex)。
2. **區塊劃分**：
   - **Header**：顯示當前總資金。
   - **左欄 (庫存監控)**：顯示各項資源的當前數量 (隨時跳動)。
   - **中欄 (生產管理)**：顯示各工廠數量，並提供「建設工廠」的按鈕與成本提示。
   - **右欄 (交易所)**：提供將各項資源賣出的按鈕與單價提示。
3. **風格**：簡潔的報表風格，按鈕需具備清晰的 `:hover` 與 `:disabled` (資金/資源不足時) 狀態。
